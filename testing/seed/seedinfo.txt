elasticdump --input ./testing/seed/assets.json --output http://localhost:9200/assets

Postgres:
- Use PG Admin/pg_dump to "Backup" the adventurelibrary database from staging. You'll need to create "advl:library567" and "rdsadmin:" users in your cluster. Then create an empty adventurelibrary database, right click and restore with the file you backedup from staging. Use the provided one in the folder to save the first step.
